# flow-forge-sdk: SDK для создания исполняемых сущностей проекта FlowForge

## Что это

В данный момент это то, с помощью чего создаются выполняемые блоки, линии, экземпляры выполняемых процедур: интерфейсы (контракты), типы данных, базовые реализации.  
В идеале - нет никаких внутренних, проектных, зависимостей.  

С помощью SDK можно создать блок своего, уникального типа, собрать его и использовать для динамического создания экземпляра типа блока в проекте FlowForge.

## Что есть и чего нет

Уже можно создавать блоки, принимающие параметры в виде текста и возвращающие результаты работы в виде текста.  
Подразумевается, что в текстовом формате можно передавать практически всё - собственно текст, json, xml, ссылки и всё, что угодно.

## Быстрый старт

```bash
mvn clean install
```

В вашем приложении, в pom.xml:

```xml
<properties>
    ...
    <flow-forge-sdk.version>2.0.5</flow-forge-sdk.version>
</properties>

<dependencies>
    <dependency>
        <groupId>ru.spb.tksoft</groupId>
        <artifactId>flow-forge-sdk</artifactId>
        <version>${flow-forge-sdk.version}</version>
    </dependency>
    ...
</dependencies>
```

Далее смотри пример реализации <ссылка на пример реализации своего уникального типа блока>
